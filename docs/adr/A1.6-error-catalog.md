# A1.6 — ADR: SOVD Error Catalog

**Status:** Accepted  
**Date:** 2025-12-01  
**Deciders:** Architecture team  
**Context:** Wave 1 — Foundation & Security

## Context

The SOVD standard (ISO 17978-3) and OData conventions require structured error
responses with machine-readable error codes. Without a centralized error catalog,
each handler invents its own error format, making client-side error handling
inconsistent and fragile.

## Decision

Define a `SovdErrorCode` enum mapping each error category to an HTTP status code
and a stable string code (e.g. `SOVD-ERR-404`). All handlers use helper functions
(`not_found()`, `bad_request()`, `conflict()`, `diag_error()`) that produce a
consistent `SovdErrorEnvelope` wrapping an OData-style error response.

```rust
pub struct SovdErrorEnvelope {
    pub error: SovdErrorResponse,
}
pub struct SovdErrorResponse {
    pub code: String,        // e.g. "SOVD-ERR-404"
    pub message: String,
    pub target: Option<String>,
    pub details: Vec<SovdErrorDetail>,
}
```

| Aspect | Decision | Rationale |
|--------|----------|-----------|
| **Error codes** | `SovdErrorCode` enum with `http_status()` and `code()` methods | Type-safe; impossible to return wrong HTTP status for a given error class |
| **Envelope** | OData `{"error": {...}}` format | ISO 17978-3 §5.2 compliance; standard for OData-based APIs |
| **Helper functions** | `not_found()`, `bad_request()`, `conflict()`, `diag_error()` | DRY; consistent error shape across 40+ handlers |
| **DiagServiceError mapping** | `diag_error()` maps backend errors to SOVD error codes | Clean separation: backend returns domain errors, HTTP layer maps to OData |

## Consequences

- **Positive:** Every error response is OData-conformant with a stable code
- **Positive:** Clients can switch on `error.code` for programmatic error handling
- **Positive:** `diag_error()` cleanly maps 12+ backend error variants to 6 HTTP status codes
- **Implementation:** `native-interfaces/src/sovd.rs` (SovdErrorCode, SovdErrorEnvelope), `native-sovd/src/routes.rs` (helpers)
