# A1.3 — ADR: Request Body Size Limit

**Status:** Accepted  
**Date:** 2025-12-01  
**Deciders:** Architecture team  
**Context:** Wave 1 — Foundation & Security

## Context

The SOVD server accepts JSON request bodies for data writes, bulk operations,
lock acquisition, and operation execution. Without a size limit, a malicious or
misconfigured client could send arbitrarily large payloads, exhausting server memory.

## Decision

Apply a global body size limit via `tower-http::limit::RequestBodyLimitLayer`,
configurable via TOML (`server.max_body_size`), default 1 MiB.

| Aspect | Decision | Rationale |
|--------|----------|-----------|
| **Mechanism** | `tower-http` middleware | Applied once at the router level; no per-handler boilerplate |
| **Default limit** | 1 MiB | Sufficient for all SOVD payloads (bulk read/write, flash initiation); prevents DoS |
| **Configuration** | `server.max_body_size` in TOML | Deployments with large flash payloads can increase the limit |
| **Error response** | 413 Payload Too Large with SOVD error envelope | Consistent with OData error format |

## Consequences

- **Positive:** Prevents memory exhaustion from oversized payloads
- **Positive:** Configurable per deployment without code changes
- **Implementation:** `native-sovd/src/routes.rs` (router layer), `native-server/src/main.rs` (config)
